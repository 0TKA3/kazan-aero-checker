# aero

Утилита для получения информации о рейсе из онлайн-расписания аэропорта Казани ([kazan.aero](https://www.kazan.aero/on-line-schedule/)).

## Требования

- [Bun](https://bun.com) v1.3+
- Установленный браузер **Firefox** (используется Selenium WebDriver)

## Установка

```bash
bun install
```

## Использование

Программа принимает два аргумента:

1. **Номер рейса** — поле `flight_number_iata` из API, как в расписании (например, `N4 705`, `SU 1234`)
2. **Дата** — поле `date` из API, в формате `DD.MM.YYYY` (например, `19.06.2026`)

```bash
bun run index.ts <номер_рейса> <дата>
```

Пример:

```bash
bun run index.ts "N4 705" 19.06.2026
```

Кавычки вокруг номера рейса обязательны, если в нём есть пробел.

### Формат аргументов

Поиск идёт по **точному совпадению** с данными API — программа не нормализует и не угадывает формат.

| Поле | Правильно | Неправильно |
|---|---|---|
| Номер рейса | `"N4 705"` | `N4705`, `N4-705`, `705` |
| Дата | `19.06.2026` | `2026-06-19`, `19/06/2026` |

В API у каждого рейса есть несколько полей с номером:

- `flight_number_iata` — **используется программой** (`N4 705`, `SU 1234`)
- `flight_number_icao` — ICAO-код (`NWS 705`, `AFL 1234`)
- `flight_number` — локальное название (`КЛ 705`, `СУ 1234`)

### Частые ошибки

**Склеенный номер рейса.** Nordwind N4 705 часто вводят как `N4705`. В API это два отдельных значения — код авиакомпании `N4` и номер `705`, разделённые пробелом.

**Неверный формат даты.** API возвращает дату как `19.06.2026`, а не `2026-06-19`. Даже если рейс существует, с неправильной датой поиск не сработает.

**Пропущены кавычки.** Без кавычек shell передаст `N4` и `705` как два отдельных аргумента:

```bash
# Неправильно — shell разобьёт аргументы
bun run index.ts N4 705 19.06.2026

# Правильно
bun run index.ts "N4 705" 19.06.2026
```

### Как узнать правильные значения

Открой [расписание на kazan.aero](https://www.kazan.aero/on-line-schedule/) или посмотри ответ API (`https://www.kazan.aero/json/dld.json`). Найди нужный рейс и скопируй значения полей `flight_number_iata` и `date` без изменений.

Пример записи из API для рейса Nordwind в Махачкалу:

```json
{
  "flight_number_iata": "N4 705",
  "date": "19.06.2026",
  "air_arr_ru": "Махачкала"
}
```

Команда для этого рейса:

```bash
bun run index.ts "N4 705" 19.06.2026
```

При успешном поиске в консоль выводится таблица с данными о рейсе:

- дата полёта
- время начала и окончания регистрации
- стойки регистрации
- выходы на посадку
- время начала и окончания посадки
- фактическое время вылета
- плановое время прибытия (местное)
- статус рейса

Если рейс не найден, выводится сообщение `Рейс не найден`.

## Примечания

- Номер рейса и дата должны **точно совпадать** с полями `flight_number_iata` и `date` в API (включая пробелы и формат `DD.MM.YYYY`).
- При запуске открывается Firefox — это нормальное поведение: через браузер загружаются данные с сайта аэропорта.
- Если аргументы не переданы, программа завершится без вывода.
- Если рейс не найден, проверь формат аргументов по таблице выше — чаще всего проблема в склеенном номере (`N4705` вместо `"N4 705"`) или дате (`2026-06-19` вместо `19.06.2026`).
